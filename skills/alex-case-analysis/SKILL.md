# /tian-case-organizer

## 概述

将目录下的中国裁判文书PDF（北大法宝等来源）批量整理为结构化Excel，包含案件基本信息和"本院认为"原文。

**输出：** `案例汇总.xlsx`（两个Sheet），保存到PDF所在目录。

---

## 工作流程

### Step 1 — 扫描目录

```bash
ls -la "<用户提供的目录>" | grep "\.pdf$"
```

列出所有PDF文件，过滤非PDF（.zip、.xlsx、.DS_Store等），向用户展示文件清单。

### Step 2 — PDF 转 Markdown

对每个PDF文件：
1. 调用 `mcp__plugin_file-converter_mineru-2md__convert_to_markdown`，参数 `file_path=<PDF路径>`, `language="ch"`
2. 轮询 `mcp__plugin_file-converter_mineru-2md__get_conversion_result`，参数 `batch_id=<返回的batch_id>`, `output_dir="<PDF所在目录>/_converted"`
3. 等待 `status == "done"`，获得 `md_paths`

所有 Markdown 输出到 `<PDF目录>/_converted/` 子目录。

如果单个PDF转换失败（超时或API错误），重试一次，仍失败则跳过并记录。

### Step 3 — 运行提取脚本

```bash
cd "<PDF所在目录>" && python3 "<本技能目录>/scripts/build_excel.py" "<PDF所在目录>/_converted" "<PDF所在目录>/案例汇总.xlsx"
```

脚本完成以下工作：
- 解析每个 `.md` 文件提取元数据和"本院认为"
- 日期标准化、审判程序简化、去重、按判决日期排序
- 生成格式化 Excel

### Step 4 — 输出汇总报告

读取脚本的输出，向用户展示：

```
案件整理完成
────────────────────────────
扫描文件：N 个 PDF
成功处理：M 个案件
去重合并：D 组重复
本院认为：P/M 个案件有完整原文
日期跨度：YYYY年MM月DD日 — YYYY年MM月DD日
输出文件：/path/to/案例汇总.xlsx
────────────────────────────
```

### Sheet 2 — 本院认为（原文）（4列）

| 列 | 标题 | 宽度 |

### 格式规范

- **标题行：** 黑体 11pt 加粗，浅蓝底纹(#D9E1F2)，水平居中，垂直居中
- **数据行：** 宋体 10pt，**所有单元格水平居中 + 垂直居中**，自动换行
- **边框：** 全表格细线边框
- **冻结：** 首行冻结
- **行高：** Sheet1 固定 100pt；Sheet2 按本院认为字数动态 60~800pt

## 提取规则

### 元数据提取

| 字段 | 提取来源 | 格式处理 |
|------|---------|---------|
| 案件名称 | `# 标题` 或正文首行 | 去HTML标签，去`#`前缀 |
| 案号 | 元数据区的 `案号：...` 或 HTML表格 | 保留原始格式 |
| 审理法院 | 元数据区 `审理法院：...` 或判决书头部 | 全称 |
| 裁判日期 | `审结日期：...` 或 `二〇XX年...` | → `2021年12月31日` 格式 |
| 审判程序 | `案件类型：民事一审` 等 | `民事一审`→`一审` |
| 原告 | `原告：XXX公司` 或 `上诉人：XXX公司` | 全称，去"住所地..." |
| 被告 | `被告：XXX公司` 或 `被上诉人：XXX公司` | 全称，去"住所地..." |
| 文书类型 | 元数据区或正文 | 判决书/裁定书 |
| 来源 | 仅从文件头部提取 | 默认"北大法宝" |

### 日期标准化

```
输入                           → 输出
2021.12.31                     → 2021年12月31日
2021.12.31 00:00:00            → 2021年12月31日
二〇二一年十二月三十一日          → 2021年12月31日
```

### 争议焦点

从"本案[的]争议焦点[在于]..."或"争议焦点："开始摘录，完整保留至句号或段落结束，不截断。

### 本院认为识别优先级

1. `本院认为：`（标准判决书段落）
2. `法院生效裁判认为`（案例报告）
3. `## 【裁判理由】`
4. `## 【裁判内容】`
5. `## 【评析】`

**结束标记（遇到即停止）：** `审判长`、`审判员`、`书记员`、`## 本案法律依据`、`© 北大法宝`、`## 【裁判要旨】`、`## 【关联索引】`、`## 同案由重要案例`

### 数据处理

- **去重：** 案号相同视为同一案件，保留元数据最完整（本院认为非空优先）的版本
- **排序：** 按裁判日期从早到晚，无日期的排最后

---

## 错误处理

| 异常 | 处理 |
|------|------|
| PDF转换失败 | 跳过，最终报告标注 |
| 无"本院认为" | 留空，案件仍纳入 |
| 缺案号/缺日期 | 纳入但标注"缺案号"/"日期待确认" |
| 空目录 | 提示"没有找到PDF文件" |
